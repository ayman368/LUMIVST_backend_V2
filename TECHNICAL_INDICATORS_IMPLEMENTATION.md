## 1. الأعمدة الجديدة في قاعدة البيانات / New Database Columns

### أ) 21-Day EMA (Exponential Moving Average)
- **اسم العمود**: `ema_21`
- **الوصف**: مؤشر المتوسط المتحرك الأسي لـ 21 يوم
- **الصيغة**: EMA = (Close - EMA_prev) × K + EMA_prev (حيث K = 2/(21+1))
- **الفائدة**: أكثر استجابة للأسعار الحديثة مقارنة بـ SMA البسيط

### ب) قيم 200MA التاريخية / Historical 200MA Values
تم إضافة 5 أعمدة جديدة لحفظ قيمة المتوسط المتحرك 200 يوم للفترات الماضية:

| العمود | الوصف |
|--------|--------|
| `sma_200_1m_ago` | قيمة 200MA قبل شهر واحد (≈ 21 يوم تداول) |
| `sma_200_2m_ago` | قيمة 200MA قبل شهرين (≈ 42 يوم تداول) |
| `sma_200_3m_ago` | قيمة 200MA قبل 3 أشهر (≈ 63 يوم تداول) |
| `sma_200_4m_ago` | قيمة 200MA قبل 4 أشهر (≈ 84 يوم تداول) |
| `sma_200_5m_ago` | قيمة 200MA قبل 5 أشهر (≈ 105 يوم تداول) |

**الفائدة**: تمكّن من رؤية هل 200MA في اتجاه صاعد (Uptrend) أم لا

### ج) المتوسطات المتحركة الأسبوعية / Weekly Moving Averages
- **`sma_30w`**: المتوسط المتحرك لـ 30 أسبوع (مشابه تقريباً لـ 150-day SMA حيث 30 × 5 = 150)
- **`sma_40w`**: المتوسط المتحرك لـ 40 أسبوع (مشابه تقريباً لـ 200-day SMA حيث 40 × 5 = 200)

**ملاحظة**: يتم حساب هذه المتوسطات بناءً على **آخر إغلاق في كل أسبوع** وليس بناءً على الأيام

## 5. الحسابات / Calculations Reference

### EMA صيغة الحساب:
```
EMA = (Close - EMA_prev) × (2 / (n + 1)) + EMA_prev
```

حيث:
- n = عدد الفترات (21 في حالتنا)
- k = 2 / (21 + 1) = 0.0909

### SMA Historical صيغة الحساب:
```
SMA_200_X_days_ago = Average(Close[today-X-days to today-X-days-199])
```

### Weekly SMA صيغة الحساب:
```
1. احصل على آخر سعر إغلاق في كل أسبوع
2. احسب المتوسط المتحرك البسيط لآخر 30 (أو 40) أسبوع
3. اعرض نفس القيمة لجميع أيام الأسبوع
```

## 7. استكشاف الأخطاء / Troubleshooting

### المشكلة: الفلاتر الجديدة لا تعمل
**الحل**: تأكد من:
1. تطبيق الهجرة بنجاح: `alembic upgrade head`
2. تشغيل سكريبت حساب المؤشرات: `python scripts/calculate_technicals.py`
3. إعادة تشغيل API

### المشكلة: قيم NULL في العمود الجديد
**الحل**: هذا طبيعي للبيانات التاريخية. سيتم ملء القيم عند:
1. حساب جميع البيانات التاريخية (قد يكون مكلفاً)
2. أو عند التحديثات اليومية الجديدة

### المشكلة: الاستعلام بطيء
**الحل**: 
1. تأكد من وجود فهارس (indexes) على الأعمدة المستخدمة في الفلترة
2. قد تحتاج لإعادة تजميع الإحصائيات: `ANALYZE prices;`
